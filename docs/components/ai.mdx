AI components in Sentris Flow follow a **Provider-Consumer** architecture. 

1.  **Providers**: Handle credentials, model selection, and API configuration (OpenAI, Gemini, Anthropic, OpenRouter).
2.  **Consumers**: Execute specific tasks (Text Generation or Autonomous Agents) using the configuration emitted by a Provider.

---

## Providers

Provider nodes normalize credentials and model settings into a reusable **LLM Provider Config**.

### OpenAI Provider

Configures access to OpenAI or OpenAI-compatible endpoints.

| Input | Type | Description |
|-------|------|-------------|
| `apiKey` | Secret | OpenAI API key (typically from a Secret Loader) |

| Parameter | Type | Description |
|-----------|------|-------------|
| `model` | Select | `gpt-5.2`, `gpt-5.1`, `gpt-5`, `gpt-5-mini` |
| `apiBaseUrl` | Text | Optional override for the API base URL |

### Gemini Provider

Configures access to Google's Gemini models.

| Input | Type | Description |
|-------|------|-------------|
| `apiKey` | Secret | Google AI API key |

| Parameter | Type | Description |
|-----------|------|-------------|
| `model` | Select | `gemini-3-pro-preview`, `gemini-3-flash-preview`, `gemini-2.5-pro` |
| `apiBaseUrl` | Text | Optional override for the API base URL |
| `projectId` | Text | Optional Google Cloud project identifier |

### Anthropic Provider

Configures access to Anthropic's Claude models.

| Input | Type | Description |
|-------|------|-------------|
| `apiKey` | Secret | Anthropic API key |

| Parameter | Type | Description |
|-----------|------|-------------|
| `model` | Select | `claude-sonnet-4-6`, `claude-opus-4-8`, `claude-haiku-4-5` |
| `apiBaseUrl` | Text | Optional override for the API base URL |

**Supported Models**

| Model | Description |
|-------|-------------|
| `claude-sonnet-4-6` | Claude Sonnet 4.6 — balanced performance and speed (default) |
| `claude-opus-4-8` | Claude Opus 4.8 — highest capability for complex tasks |
| `claude-haiku-4-5` | Claude Haiku 4.5 — fastest and most cost-effective |

**Environment Variables**

| Variable | Description |
|----------|-------------|
| `ANTHROPIC_BASE_URL` | Default base URL override |

### OpenRouter Provider

Configures access to multiple LLM providers through OpenRouter's unified API.

| Input | Type | Description |
|-------|------|-------------|
| `apiKey` | Secret | OpenRouter API key |

| Parameter | Type | Description |
|-----------|------|-------------|
| `model` | Text | Model slug (e.g., `openrouter/auto`, `anthropic/claude-3.5-sonnet`) |
| `apiBaseUrl` | Text | Optional override for the API base URL |
| `httpReferer` | Text | Application URL for OpenRouter ranking |
| `appTitle` | Text | Application title for OpenRouter ranking |

## Consumers

Consumer nodes perform the actual AI work. They require a **Provider Config** output from one of the providers above.

### AI Generate Text

Performs a one-shot chat completion.

| Input | Type | Description |
|-------|------|-------------|
| `userPrompt` | Text | The primary request or data to process |
| `chatModel` | Credential | **Required.** Connect a Provider output here |
| `modelApiKey` | Secret | Optional. supersedes the API key in the Provider Config |

| Parameter | Type | Description |
|-----------|------|-------------|
| `systemPrompt`| Textarea | Instructions that guide the model's behavior |
| `temperature` | Number | Creativity vs. determinism (0.0 to 2.0) |
| `maxTokens` | Number | Maximum tokens to generate |

| Output | Type | Description |
|--------|------|-------------|
| `responseText`| Text | The assistant's response |
| `usage` | JSON | Token consumption metadata |
| `rawResponse` | JSON | Full API response for debugging |

### AI SDK Agent

An autonomous agent that uses reasoning steps and tool-calling to solve complex tasks.

| Input | Type | Description |
|-------|------|-------------|
| `userInput` | Text | The task or question for the agent |
| `chatModel` | Credential | **Required.** Connect a Provider output here |
| `conversationState` | JSON | Optional. Connect from a previous turn for memory |
| `mcpTools` | List | Optional. Connect tools from MCP Providers |

| Parameter | Type | Description |
|-----------|------|-------------|
| `systemPrompt`| Textarea | Core identity and constraints for the agent |
| `temperature` | Number | Reasoning creativity (default 0.7) |
| `stepLimit` | Number | Max "Think -> Act -> Observe" loops (1-12) |
| `memorySize` | Number | Number of previous turns to retain in context |
| `structuredOutputEnabled` | Toggle | Enable to enforce a specific JSON output structure |
| `schemaType` | Select | How to define the schema: `json-example` or `json-schema` |
| `jsonExample` | JSON | Example JSON object for schema inference (all properties become required) |
| `jsonSchema` | JSON | Full JSON Schema definition for precise validation |
| `autoFixFormat` | Toggle | Attempt to extract valid JSON from malformed responses |

| Output | Type | Description |
|--------|------|-------------|
| `responseText`| Text | Final answer after reasoning is complete |
| `structuredOutput` | JSON | Parsed structured output (when enabled) |
| `conversationState` | JSON | Updated state to pass to the next agent node |
| `reasoningTrace` | JSON | Detailed step-by-step logs of the agent's thoughts |
| `agentRunId` | Text | Unique session ID for tracking and streaming |

### OpenCode Agent

An autonomous coding and investigation agent powered by [OpenCode](https://github.com/opencode-ai/opencode). Runs in an isolated Docker container with access to connected MCP tools for deep security investigations.

| Input | Type | Description |
|-------|------|-------------|
| `task` | Text | The investigation task or question for the agent |
| `context` | JSON | Optional contextual data (alerts, logs, previous findings) |
| `model` | Credential | **Required.** Connect a Provider output here |
| `tools` | List | Optional. Connect tool-mode nodes to expose them to the agent |
| `supplementaryInputA` | Text | Optional scanner or artifact text written to `/workspace/supplementary-a.txt` |
| `supplementaryInputB` | Text | Optional additional text written to `/workspace/supplementary-b.txt` |

| Parameter | Type | Description |
|-----------|------|-------------|
| `systemPrompt` | Textarea | Override the default investigator prompt template |
| `autoApprove` | Toggle | Automatically approve agent actions (default: true). Maps to OpenCode `permission: allow` vs `ask` |
| `providerConfig` | JSON | Additional OpenCode provider configuration |
| `skillIds` | Multi-select | Org [Agent Skills](/agent-skills) injected under `.opencode/skills/{slug}/SKILL.md` |
| `enablePlugins` | Multi-select | Optional plugins: `oh-my-claudecode`, `superpowers` (when pre-installed in the image) |

| Output | Type | Description |
|--------|------|-------------|
| `report` | Text | Final markdown report generated by the agent |
| `rawOutput` | Text | Full stdout/stderr logs from the agent execution |

**Example use cases:**
- Autonomous investigation of security alerts using connected threat intel tools
- Code analysis and vulnerability assessment with MCP-connected scanners
- Multi-step research tasks that require tool-calling and reasoning

### Claude Code Agent

An autonomous investigation agent powered by the official [Claude Code CLI](https://www.npmjs.com/package/@anthropic-ai/claude-code). Runs headlessly in Docker with Sentris MCP gateway tools and org Agent Skills.

| Input | Type | Description |
|-------|------|-------------|
| `task` | Text | The investigation task or question for the agent |
| `context` | JSON | Optional contextual data (alerts, logs, previous findings) |
| `model` | Credential | **Required.** Connect a Provider output (Anthropic recommended) |
| `tools` | List | Optional. Connect tool-mode nodes to expose them to the agent |
| `supplementaryInputA` | Text | Optional text written to `/workspace/supplementary-a.txt` |
| `supplementaryInputB` | Text | Optional text written to `/workspace/supplementary-b.txt` |

| Parameter | Type | Description |
|-----------|------|-------------|
| `systemPrompt` | Textarea | Override the default investigator prompt template |
| `autoApprove` | Toggle | Skip permission prompts when true (`--dangerously-skip-permissions`) |
| `providerConfig` | JSON | Additional settings merged into `settings.json` |
| `skillIds` | Multi-select | Org [Agent Skills](/agent-skills) injected under `.claude/skills/{slug}/SKILL.md` |
| `enablePlugins` | Multi-select | Optional plugins from `/opt/plugins/` in the Docker image |

| Output | Type | Description |
|--------|------|-------------|
| `report` | Text | Final markdown report generated by the agent |
| `rawOutput` | Text | Full stdout/stderr logs from the agent execution |

**Docker image:** `ghcr.io/zebbern/claude-code:latest` (see `docker/claude-code/Dockerfile`)

Unlike OpenCode, Claude Code Agent **fails the workflow step** on non-zero CLI exit codes.

#### Authentication

Claude Code supports two inline authentication modes on the **Model & Authentication** panel (or via a connected Provider node):

| Mode | Billing | Secret contents | Env var at runtime |
|------|---------|-----------------|-------------------|
| **API Key** | Pay-as-you-go Anthropic API | `sk-ant-...` API key | `ANTHROPIC_API_KEY` |
| **Subscription (Pro/Max)** | Claude subscription | Output of `claude setup-token` | `CLAUDE_CODE_OAUTH_TOKEN` |

**Subscription setup:**

1. On a machine where you are logged in to Claude (`claude login`), run:
   ```bash
   claude setup-token
   ```
2. Create a secret in **Manage → Secrets** and paste the token value.
3. In the Claude Code node, set **Authentication** to **Subscription (Pro/Max)** and select that secret.
4. Save the workflow before running.

**Rotation:** Re-run `claude setup-token`, then rotate the secret value in Manage → Secrets. The subscription token is inference-scoped; use API Key mode for API-billed usage.

Sentris never sets both `ANTHROPIC_API_KEY` and `CLAUDE_CODE_OAUTH_TOKEN` on the same run.

#### Model selection

The **Model & Authentication** panel lets you pick the model the agent runs with (passed to the CLI as `ANTHROPIC_MODEL`):

- In **API Key** mode, Sentris fetches the live list of models your key can access (Anthropic `GET /v1/models`) and merges it with a curated fallback. Use **Refresh List** to re-fetch.
- In **Subscription (Pro/Max)** mode, Anthropic does not allow listing models with OAuth tokens, so a curated list is shown.
- You can always type an **exact model id** (e.g. `claude-opus-4-8`) to use a model that is not listed.

#### Effort level

The **Effort level** control sets how many tokens Claude spends (capability vs speed/cost), matching Anthropic's [effort parameter](https://platform.claude.com/docs/en/build-with-claude/effort):

| Level | Behavior |
|-------|----------|
| Default (model default) | No override — runs at full default capability (high) |
| Low | Fastest and cheapest; simpler tasks |
| Medium | Balanced speed, cost, and quality |
| High | High capability (API default) |
| Extra high (xhigh) | Deep, long-horizon agentic and coding work |
| Max | Absolute maximum capability, no token constraints |

When a non-default level is selected, Sentris sets `CLAUDE_CODE_EFFORT_LEVEL` in the container (the reliable headless mechanism, including for `max`). When left at Default, no override is applied so the agent performs at its best by default.

## MCP Tools (Model Context Protocol)

Sentris Flow supports the **Model Context Protocol (MCP)**, allowing AI agents to interact with external tools over HTTP.

### MCP HTTP Tools

Exposes a set of tools from a remote HTTP server that implements the MCP contract.

| Input | Type | Description |
|-------|------|-------------|
| `endpoint` | Text | The HTTP URL where the MCP server is hosted |
| `headersJson` | Text | Optional JSON of headers (e.g., Auth tokens) |
| `tools` | JSON | List of tool definitions available on that endpoint |

| Parameter | Type | Description |
|-----------|------|-------------|
| `endpoint` | Text | Destination URL for tool execution |
| `tools` | JSON | Array of tools with `id`, `title`, and `arguments` |

| Output | Type | Description |
|--------|------|-------------|
| `tools` | List | Normalized MCP tool definitions for the AI Agent |

### MCP Tool Merge

Combines multiple MCP tool lists into a single consolidated list.

| Input | Type | Description |
|-------|------|-------------|
| `toolsA`, `toolsB`| List | Multiple upstream MCP tool outputs |

| Parameter | Type | Description |
|-----------|------|-------------|
| `slots` | JSON | Configure additional input ports for merging |

| Output | Type | Description |
|--------|------|-------------|
| `tools` | List | De-duplicated list of tools ready for an agent |

## Use Cases

### Automated Alert Triage
**Flow:** `Provider` → `AI Generate Text`

Analyze incoming security alerts to filter out false positives.
**Prompt:** "Given this alert payload: {{alert}}, determine if it's a real threat or noise."

### Investigative Agent
**Flow:** `Provider` + `MCP Tool` → `AI Agent`

An agent that searches through logs and performs lookups to investigate a specific IP address.
**Task:** "Investigate the IP {{ip}} using the available Splunk and VirusTotal tools."

### Structured Output for Data Extraction
**Flow:** `Provider` → `AI Agent` (with Structured Output enabled)

Extract structured data from unstructured security reports. Enable **Structured Output** and provide a JSON example:
```json
{
  "severity": "high",
  "affected_systems": ["web-server-01"],
  "remediation_steps": ["Patch CVE-2024-1234", "Restart service"]
}
```
The agent will always return validated JSON matching this schema, ready for downstream processing.

## Best Practices

<Note>
  **The Provider Concept**: Always place a Provider node (OpenAI/Gemini/Anthropic/OpenRouter) at the start of your AI chain. This allows you to swap models or providers for the entire workflow by changing just one node.
</Note>

### Prompt Engineering
1.  **Use Structured Output**: When you need consistent JSON for downstream nodes, enable **Structured Output** instead of relying on prompt instructions. This guarantees schema compliance and eliminates parsing errors.
2.  **Use System Prompts**: Set high-level rules (e.g., "You are a senior security researcher") in the System Prompt parameter instead of the User Input.
3.  **Variable Injection**: Use `{{variableName}}` syntax to inject data from upstream nodes into your prompts.

### Memory & State
For multi-turn conversations, always loop the `conversationState` output of the AI Agent back into the `conversationState` input of the next agent invocation (or store it in a persistent variable).
