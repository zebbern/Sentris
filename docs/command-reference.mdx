Sentris Flow uses [just](https://github.com/casey/just) as its command runner. Run `just` or `just help` to see all available commands.

---

## Getting Started

### Initialize Project

```bash
just init
```

Sets up the development environment:
- Installs dependencies with Bun
- Creates environment files from examples (`backend/.env`, `worker/.env`, `frontend/.env`)

---

## Development Commands

### Start Development Environment

```bash
just dev
```

Starts the full development environment with hot-reload:
- Checks for environment files
- Starts Docker infrastructure (Postgres, Temporal, MinIO, Redis, Loki, OpenSearch, Redpanda)
- Waits for services to be ready
- Runs database migrations
- Starts backend, worker, and frontend via PM2

**Auth mode auto-detection:** If `CLERK_SECRET_KEY` is set in `backend/.env`, secure mode is enabled (Clerk auth + OpenSearch Security). Otherwise, local auth mode is used.

**Access points (all via nginx on port 80):**
| Service | URL |
|---------|-----|
| Frontend | http://localhost/ |
| Backend API | http://localhost/api/ |
| Analytics | http://localhost/analytics/ |
| Temporal UI | http://localhost:8081 |

### Stop Development

```bash
just dev stop
```

Stops all PM2 processes and Docker infrastructure.

### View Logs

```bash
just dev logs
```

Streams logs from all application processes via PM2.

### Check Status

```bash
just dev status
```

Shows status of PM2 processes and Docker containers.

### Clean Development

<Danger>This deletes all data.</Danger>

```bash
just dev clean
```

Stops PM2 processes, removes Docker infrastructure and volumes.

## Production Commands

### Start Production

```bash
just prod
```

Starts the production environment using Docker Compose. Auto-detects security mode: if TLS certs exist in `docker/certs/`, secure mode with multitenancy is enabled.

**Access points (all via nginx on port 80):**
| Service | URL |
|---------|-----|
| Frontend | http://localhost/ |
| Backend API | http://localhost/api/ |
| Analytics | http://localhost/analytics/ |
| Temporal UI | http://localhost:8081 |

### Start Latest Release

```bash
just prod start-latest
```

Fetches the latest release from GitHub, pulls matching GHCR images, and starts production.

### Build and Start

```bash
just prod build
```

Builds Docker images and starts the production environment.

### Stop Production

```bash
just prod stop
```

Stops all production containers.

### View Production Logs

```bash
just prod logs
```

Streams logs from all production containers.

### Check Production Status

```bash
just prod status
```

Shows status of production containers.

### Clean Production

<Danger>This deletes all data.</Danger>

```bash
just prod clean
```

Stops containers, removes volumes, and prunes Docker system.

## Infrastructure Commands

### Start Infrastructure

```bash
just infra up
```

Starts only the infrastructure containers:
- PostgreSQL
- Temporal
- MinIO
- Redis
- Loki
- OpenSearch
- Redpanda

### Stop Infrastructure

```bash
just infra down
```

Stops all infrastructure containers.

### View Infrastructure Logs

```bash
just infra logs
```

Streams logs from infrastructure containers.

### Clean Infrastructure

```bash
just infra clean
```

Stops infrastructure and removes all volumes. 

<Danger>This deletes all data.</Danger>

## Utility Commands

### Show All Status

```bash
just status
```

Comprehensive status check showing:
- PM2 services
- Infrastructure containers
- Production containers

### Reset Database

```bash
just db-reset
```

Drops and recreates the database, then runs migrations. **This deletes all data.**

<Warning>
  This command requires PostgreSQL to be running (`just dev` or `just infra up`).
</Warning>

### Build Images Only

```bash
just build
```

Builds production Docker images without starting containers.

## GHCR Image Commands

### Start with GHCR Images

```bash
just prod-images
```

Pulls prebuilt images from GHCR and starts the production environment.

### Build Test Images

```bash
just prod-images build-test
```

Builds production images with PostHog analytics baked in. Requires `POSTHOG_API_KEY` and `POSTHOG_HOST` environment variables.

### Stop / Logs / Status / Clean

```bash
just prod-images stop
just prod-images logs
just prod-images status
just prod-images clean
```

Manage the GHCR-based production environment.

## Security Commands

### Generate TLS Certificates

```bash
just generate-certs
```

Generates TLS certificates for OpenSearch in `docker/certs/`. Required before running production in secure mode.

### Initialize OpenSearch Security

```bash
just security-init
just security-init --force
```

Initializes or reinitializes the OpenSearch Security index. Use `--force` to update an existing configuration.

### Hash Password

```bash
just hash-password
just hash-password 'my-password'
```

Generates a BCrypt hash for OpenSearch internal user passwords.

## Multi-Instance Commands

### Show Active Instance

```bash
just instance show
```

Prints the currently active instance number (stored in `.sentris-instance`).

### Set Active Instance

```bash
just instance use <0-9>
```

Sets the active instance for the current workspace. All subsequent `just dev` commands target this instance.

### Initialize Instance Environment

```bash
just instance-init [N]
```

Creates instance-specific env files at `.instances/instance-N/`. Never overwrites existing files.

### Manage Instance Env Files

```bash
just instance-env init [N] [--force]   # Generate env files
just instance-env update [N]           # Patch instance-specific vars
just instance-env copy SRC DEST        # Copy env between instances
just instance-env show [N]             # Show instance config
```

## Environment Files

The `just init` command creates these environment files:

| File | Purpose |
|------|---------|
| `backend/.env` | Backend API configuration |
| `worker/.env` | Worker configuration |
| `frontend/.env` | Frontend configuration |

Each file is created from its corresponding `.env.example` template.
